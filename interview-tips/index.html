Coding Interviews

As of 2022/04, many companies do some form of data structures and algorithm interview.

In my experience, the algorithm part is there to evaluate one’s problem-solving ability and familiarity with core software engineering concepts that are needed for most coding roles. Aside from that, consider the aspects that you would apply to a coding problem you tackle at work. Namely:

  • share your thought process: how did you arrive at your conclusion? what are the tradeoffs?
  • code design: how would you write this code so the next engineer can understand it?
  • testing: how would you test this code? what are the edge cases?

For coding interviews, getting into the rhythm and being able to type up answers quickly is key. I recommend leetcode medium, as well as running through cracking the coding interview.

Design Interviews

For design interviews, I recommend reading up on Grokking the system design interview.

Tips include:

  • Have a checklist memorized to run through the basics like
    • verifying assumptions
    • estimating

Behavioral Questions

  • Have 5-6 stories around your experiences in the STAR format.
  • Have some stories around projects that didn’t go as well as expected. Showing reflection and growth is important.
